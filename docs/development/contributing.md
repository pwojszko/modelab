# Contributing

Thank you for your interest in contributing to ModelSLab!

## How to Contribute

### Reporting Bugs

If you found a bug:

1. Check if the problem has not already been reported
2. Create a new issue with a description of the problem
3. Include steps to reproduce
4. Describe expected vs actual behavior

### Proposing Features

Have an idea for a new feature?

1. Create an issue with a description of the feature
2. Explain why the feature would be useful
3. Describe the proposed implementation (if you have an idea)

### Submitting Pull Requests

1. **Fork** the repository
2. Create a **branch** for your change:
   ```bash
   git checkout -b feature/feature-name
   ```
3. Make **changes** and add **tests** (if possible)
4. Make sure the code **works** and **compiles**
5. **Commit** changes with a descriptive message:
   ```bash
   git commit -m "Add feature X"
   ```
6. **Push** to your fork:
   ```bash
   git push origin feature/feature-name
   ```
7. Create a **Pull Request**

## Code Standards

### Python

- Use **PEP 8** as style guidelines
- Add **docstrings** to functions and classes
- Use **type hints** where possible
- Maximum line length: **88 characters** (Black)

### C++

- Use **snake_case** for functions
- Add **Doxygen comments** to functions
- Maintain consistency with existing code
- Maximum line length: **100 characters**

### Markdown (documentation)

- Use **Markdown** according to MkDocs conventions
- Add **code examples** where possible
- Maintain **consistent style** with the rest of the documentation

## Review Process

1. All PRs are reviewed
2. We may ask for changes
3. After acceptance, the PR will be merged

## Questions?

If you have questions, create an issue or contact the project maintainers.
